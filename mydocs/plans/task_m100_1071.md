## 3. 문제 정의

이슈 #1067에서 도형 회전과 도형 자체 출력은 정정되었지만, 다음 두 문제가 남아 있다.

```text
1. HWPX TAC 도형이 paragraph layout 에서 HWP 정답지보다 가로로 더 붙어 보인다.
2. 도형 컨트롤이 문서 편집 시 한글자처럼 캐럿 이동되지 않는다.
```

현재 이슈 본문에 기록된 좌표 차이는 다음과 같다.

```text
HWP SVG:
  shape1 cx=132
  shape2 cx=156.5
  gap=24.5px

HWPX SVG:
  shape1 cx=126
  shape2 cx=137.5
  gap=11.5px
```

HWP/HWPX IR은 동일하게 보이므로, 이번 문제는 parser layer 보다는 `paragraph_layout`의
TAC control advance 계산, 공백 문자 폭 반영, control marker/char offset 처리 쪽을 우선 의심한다.

## 4. 범위

### 포함

- `shape-001` HWP/HWPX 의 paragraph layout 좌표 차이 재현
- TAC 도형과 공백 문자 advance 계산 비교
- HWP/HWPX 동일 IR에서 layout 결과가 갈라지는 지점 특정
- SVG export 또는 layout dump 기반 회귀 가드 추가
- rhwp-studio 커서 이동 처리에서 TAC 도형 컨트롤을 한글자처럼 이동시키는 수정
- 필요 시 Rust `char_offsets`, control text position, web cursor 상태 동기화 정정

### 제외

- 도형 회전 재정정
- HWPX 저장기능 전반 재검토
- textWrap 일반 도형 배치 확장
- shape-001과 무관한 광범위 paragraph layout 재작성

## 5. 단계 계획

### Stage 1 — layout 좌표 차이 재현과 원인 특정

목표는 구현 전에 HWP/HWPX가 같은 IR을 가진 상태에서 왜 다른 x 좌표로 배치되는지 정확히 찾는 것이다.

작업:

```text
1. 기존 issue_1067 SVG와 새 SVG export를 비교해 shape-001 좌표 차이를 재확인한다.
2. 해당 문단의 text, controls, char_offsets, char_shapes, line_segs, ParaShape를 dump한다.
3. paragraph_layout에서 TAC 도형이 line item으로 들어가는 경로를 추적한다.
4. 도형 control marker, space 문자, advance width 누적값을 HWP/HWPX 각각 비교한다.
5. 원인 후보를 "공백 폭", "control advance", "x origin", "control marker width", "scale/quantize" 중 하나로 좁힌다.
```

산출물:

```text
mydocs/working/task_m100_1071_stage1.md
output/poc/task1071/stage1_tac_shape_layout_trace/
```

### Stage 2 — TAC 도형 + 공백 가로 배치 정정

Stage 1에서 특정한 계산 지점만 핀셋 수정한다.

작업:

```text
1. HWP/HWPX 동일 IR이면 동일한 paragraph line item sequence와 x advance가 나오도록 수정한다.
2. TAC 도형 자체 폭과 도형 전후 space 문자 폭이 모두 배치에 반영되도록 한다.
3. shape-001 HWP/HWPX SVG 좌표 차이를 회귀 가드로 고정한다.
4. 기존 shape rotation 테스트와 paragraph layout 테스트를 함께 실행한다.
```

산출물:

```text

```

### Stage 3 — 도형 컨트롤 한글자 캐럿 이동 정정

도형 컨트롤은 편집 시 한 글자처럼 취급되어야 한다. 줄 끝이나 문단 끝이 도형인 경우에도
캐럿이 도형 다음 위치로 이동해야 한다.

작업:

```text
1. rhwp-studio cursor.ts, input-handler, selection/caret 이동 경로를 확인한다.
2. TAC 도형 control marker가 char_offset 단위 이동 대상에 포함되는지 확인한다.
3. 오른쪽/왼쪽 방향키 이동 시 도형 앞/뒤 위치로 정상 이동하도록 수정한다.
4. HWP shape-001.hwp 와 HWPX shape-001.hwpx 양쪽에서 커서 이동을 검증한다.
```

산출물:

```text

output/poc/task1071/stage3_tac_shape_cursor/
```
