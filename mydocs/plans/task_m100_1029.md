### 2.2 메커니즘

PR #1003 의 merge 메시지에 명시:
> "옵션 A: 3 본질 커밋 cherry-pick (**d7663dd4 Stage 2 layout.rs PR #1005 영역 `--theirs` 충돌 해소** ...)"

`d7663dd4` cherry-pick 시 PR #1011 영역과 충돌 → `--theirs` 해소로 PR #1011 의 `PageBorderBasis` 기반 로직이 PR #987 시절 `attr & 0x01` 비트 로직으로 revert.

해당 diff (revert 된 변경):
```diff
-        use crate::model::page::PageBorderBasis;
-        let paper_based = matches!(pbf.basis, PageBorderBasis::PaperBased);
+        let paper_based = (pbf.attr & 0x01) != 0;
```

### 2.3 포맷별 attr 단언 (RHWP_DEBUG_PAGE_BORDER=1 측정)

```
HWP3 native   : attr=0x00000000 bit0=0 → paper_based=false → body-edge (회귀)
HWP5 변환본   : attr=0x00000001 bit0=1 → paper_based=true  → paper-edge (OK)
HWPX 변환본   : attr=0x00000041 bit0=1 → paper_based=true  → paper-edge (OK)
```

HWP3 의 PageBorderFill 데이터는 attr=0 → 회귀 로직에서 paper_based=false → body-edge. PR #1011 의 `basis` 필드는 모든 포맷 parser 에서 `PaperBased` 로 명시적 주입했기 때문에 attr 비트 0 과 무관했으나, revert 로 통합 깨짐.

## 3. Fix 방향

### 3.1 옵션 A (권고): `basis` 필드 기반 로직 재복원

`src/renderer/layout.rs` 의 PageBorder 그리기 분기 2 곳 모두 PR #1011 의 `PageBorderBasis::PaperBased` 매칭으로 재복원.

- **장점**:
  - PR #1011 의 정합 contract 재확립
  - HWP5/HWPX 변환본 무영향 (이미 paper_based=true)
  - PR #1003 Task #990 본질 (treat-as-char 글상자 advance 이중 가산) 과 직교 — 영향 없음
- **단점**: 없음
- **회귀 risk**: 0 — PR #1011 의 sweep 결과 동일하게 도달

### 3.2 옵션 B: HWP3 parser 에서 attr bit 0 = 1 주입

`src/parser/hwp3/mod.rs` 에서 PageBorderFill 의 attr 에 bit 0 set.

- **장점**: layout.rs 무수정
- **단점**: attr 의 의미 변경 (HWP3 원본 데이터 attr=0 을 1 로 위조) → 보존성 위반, dump/IR 비교 정합 깨짐
- **권장 안 함**

→ **결정: 옵션 A**
