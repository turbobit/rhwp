# Task #994 수행 계획서 — HWP5 line_segs 누락 paragraph 의 겹침 해소

- 이슈: [#994](https://github.com/edwardkim/rhwp/issues/994)
- 브랜치: `local/task994` (base: `local/devel`)
- 마일스톤: v1.0.0

## 1. 배경

`samples/hwp3-sample16-hwp5.hwp` 페이지 19 부터 paragraph 들이 시각적 겹침. 페이지 18 까지 정상.

동일 원본의 HWP3 (`samples/hwp3-sample16.hwp`) 는 모든 페이지 정상.

## 2. Root cause (확정)

### IR 패턴 분석

| Paragraph 타입 | HWP3 line_segs | HWP5 line_segs |
|---------------|----------------|----------------|
| `󰏅 ...` (PUA bullet, long text) | 있음 | **누락** |
| ` ◦ ...` (regular bullet) | 있음 | 있음 |

→ Hancom HWP3 → HWP5 변환 시 일부 paragraph 의 `PARA_LINE_SEG` 레코드 누락.

### 코드 영향

[src/renderer/composer.rs:301-326](src/renderer/composer.rs#L301-L326):
```rust
if para.line_segs.is_empty() {
    return vec![ComposedLine {
        line_height: 400,  // 400 HU = 5.33 px — 너무 작음!
        baseline_distance: 320,
        ...
    }];
}
```

→ 165 char 짜리 텍스트가 line_height=5.33px 의 단일 라인 → wrap line 들이 같은 y 좌표에 겹침.
