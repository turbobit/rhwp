## 1. 증상

`samples/hwp3-sample18.hwp` (HWP3) 가 rhwp 에서 **69 페이지** 인식, 한컴 viewer **67 페이지** — **+2 페이지 inflate**.

## 2. 페이지 비교

| Format | rhwp | 한컴 | 차이 |
|--------|------|------|------|
| HWP3 (.hwp) | 69 | 67 | **+2** |
| HWP5 변환본 (.hwp) | 67 | 67 | 0 ✓ |
| HWPX 변환본 (.hwpx) | 74 | 67 | +7 |

→ HWP3 + HWPX 만 inflate. HWP5 변환본은 정합.

## 3. Stage 1 진단 결과 (사전 확인)

`dump-pages` 분석:
- **Page 2**: pi=27 "(빈)" 1개만, vpos=69356 HU (= 925 px ≈ body height 935 px 근접)
- **Page 14**: pi=164 "(빈)" 1개만, vpos=69836 HU (= 931 px)

→ 빈 paragraph 가 별도 page 차지 → +2 inflate.

## 4. 추정 root cause

### 후보 A — typeset.rs 의 vpos-reset 처리
빈 paragraph 의 vpos 가 이전 page 의 body bottom 근처 일 때, typeset 가 새 page 로 분기.

### 후보 B — HWP3 parser 의 line_segs vpos 인코딩
HWP3 의 line_segs.vertical_pos 값이 잘못 계산되어 page 끝을 넘어가는 것으로 해석.

### 후보 C — empty paragraph 의 line_height 누적
빈 paragraph 의 line_height + spacing 이 이전 page 의 잔여 공간을 초과하는 것으로 계산.

### 후보 D — Page break trigger 조건 (next_will_vpos_reset 등)
typeset.rs 의 next paragraph vpos-reset 가드가 잘못 trigger.
