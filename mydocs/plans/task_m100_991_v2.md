## v1 정정 사유

v1은 "글자처럼 취급 표가 빈 문단에 단독 앵커되면 렌더링 누락"으로 진단했으나, 이는 **조사자의 SVG 텍스트 추출 스크립트 결함**(셀 텍스트 색상 `#5d5d5d`를 `#000000` 필터로 누락)에 의한 오진이었다. 표는 정상 렌더링된다.

1단계 조사에서 올바르게 재추출하여 **실제 버그**를 확인했다.

## 실제 문제 정의

`treat_as_char` 표가 쪽 경계에서 분할될 때, 셀 내부 한 문단의 줄이 **분할 경계 양쪽 페이지에 중복 렌더링**된다.

확인 사례(비공개 샘플, 섹션 0 문단 103, 1행×1열 표):

| 줄 | 6쪽(분할 끝) | 7쪽(분할 시작) |
|----|-------------|---------------|
| cp[0] ☞ 노후 시스템… | O | — |
| cp[1] ☞ 사용자 편의성… | **O** | **O** (중복) |
| cp[2]~cp[4] | — | O |

## 근본 원인

`src/renderer/layout/table_layout.rs` 의 `compute_cell_line_ranges` 가 분할 경계 양쪽에서 **비대칭 기준**을 사용한다.

- 분할 끝 페이지(`content_limit`): 줄 포함 판정에 `line_break_pos = cum + h` 사용 — 줄간격(trail line-spacing) **제외** (Task #656 의도된 모델).
- 분할 시작 페이지(`content_offset`): 줄 스킵 판정에 `line_end_pos = cum + line_h` 사용 — 줄간격 **포함**.

분할 경계가 한 줄의 줄간격 구간 `[cum+h, cum+h+ls]` 안에 떨어지면, 끝 페이지는 `break_pos ≤ 경계`로 포함하고 시작 페이지는 `end_pos > 경계`로도 포함 → 중복.

Task #656 이 끝 페이지 측만 `cum+h` 모델로 전환하고 시작 페이지(offset 스킵) 측을 함께 갱신하지 않은 누락으로 보인다.
