# 수행 계획서: Task M100-1122

- 이슈: [#1122](https://github.com/edwardkim/rhwp/issues/1122)
- 마일스톤: M100 / v1.0.0
- 브랜치: `local/task1122-equation-over-token`
- 대상 파일: `samples/3-11월_실전_통합_2022.hwp`
- 대상 화면: page 4 (0-index page 3), `http://localhost:7700` / VS Code extension 계열 렌더

## 1. 증상

작업지시자 제공 캡처 기준으로, 한컴오피스 2022에서는 page 4의 보기 수식이 분수 형태로 보이지만 rhwp 렌더에서는 일부 수식이 `11over20`, `3over5`, `13over20`, `7over10`, `3over4`처럼 한 줄 텍스트에 가깝게 표시된다.

추가로 문 26)의 주머니 그림이 한컴오피스보다 매우 좁게 렌더링되어 보기 영역과 그림 배치가 달라진다. 작업지시자 확인에 따라 이 문제도 별도 이슈를 만들지 않고 #1122 범위에서 함께 처리한다.

## 2. 초기 재현

소스 수정 없이 다음 명령으로 page 4를 확인했다.

```bash
./target/release/rhwp dump-pages samples/3-11월_실전_통합_2022.hwp -p 3
./target/release/rhwp export-svg samples/3-11월_실전_통합_2022.hwp -p 3 --debug-overlay -o output/task1121_probe
./target/release/rhwp dump samples/3-11월_실전_통합_2022.hwp -s 0 -p 223
./target/release/rhwp dump samples/3-11월_실전_통합_2022.hwp -s 0 -p 224
./target/release/rhwp dump samples/3-11월_실전_통합_2022.hwp -s 0 -p 225
```

확인 결과:

- `pi=214~215`의 `{8} over {13}`, `{17} over {26}` 등 공백이 있는 분수 스크립트는 SVG에서 분수선과 분모가 출력된다.
- `pi=223~225`의 `11 over20`, `3 over5`, `13 over20`, `7 over10`, `3 over4`처럼 `over` 뒤에 공백이 없는 스크립트는 SVG에서 `over20` 등이 이탤릭 텍스트로 출력된다.
- 따라서 HWP5 파싱 단계에서 수식 스크립트가 사라진 문제보다는, 수식 토크나이저가 `over20`을 `over` + `20`으로 분리하지 않는 문제가 1차 후보이다.
- 문 26) 주머니 그림은 `common=3365x9446 HU`, `current=9014x9446 HU`로 기록되어 있으나 SVG 렌더 경로가 `common.width`를 사용해 폭 44.86px로 출력한다. 한컴오피스 표시 폭은 `current_width`에 가까워 picture 표시 크기 선택 경로도 후보이다.

## 3. 관련 기존 이력

- Task #505 / PR #507 계열: `OVER`/`ATOP` 중위 연산자 처리 정정.
- Task #960 / PR #963: 같은 샘플 page 2의 multi-line equation line 매핑 정정.
- Task #962 / PR #964: 같은 샘플 page 2의 글상자 내부 inline equation duplicate emit 차단.

- Task #576: `times`/`sim` 키워드 prefix 분리. 현재 토크나이저 주석에는 `alpha/over/sqrt`는 항상 공백 구분된다고 되어 있으나, 이번 샘플에서 `over20` 반례가 확인됐다.

## 4. 단계 계획

1. Stage 1: 최소 재현 테스트 작성
   - `parse("11 over20")`가 `Fraction(11, 20)`이 되어야 함을 단위 테스트로 고정한다.
   - `parse("3 over5")`, `parse("7 over10")`, `parse("{8} over {13}")`도 함께 확인한다.
2. Stage 2: 토크나이저 키워드 분리 정정
   - `src/renderer/equation/tokenizer.rs::read_command`에서 `over`/`OVER`와 `atop`/`ATOP`이 숫자와 붙은 경우를 명령어로 분리한다.
   - 이미 존재하는 `times`/`sim` prefix 분리와 같은 좁은 규칙으로 제한한다.
3. Stage 3: 렌더 재검증
   - `output/task1121_after/3-11월_실전_통합_2022_004.svg`를 생성해 `over20` 텍스트가 사라지고 분수선이 생성되는지 확인한다.
4. Stage 4: 회귀 검증
   - 수식 파서/토크나이저 단위 테스트.
   - 기존 Task #505 테스트.
   - 시험지 page 4 SVG 점검.
   - 필요 시 `exam_math.hwp`, `math-001.hwp`, `eq-01.hwp`의 대표 수식 회귀 확인.
5. Stage 5: 문 26) 그림 표시 크기 정정
   - picture layout/render 경로에서 `shape_attr.current_width/current_height`가 `common.width/height`보다 큰 경우 현재 표시 크기를 사용한다.
   - 기존 shape overflow 계산과 같은 방향으로 `current` 크기를 우선한다.
6. Stage 6: 단계 보고서와 최종 보고서 작성 후 커밋/PR 준비.

## 5. 위험 평가

| 위험 | 평가 | 대응 |
|------|------|------|
| `overlap`, `overline` 같은 LaTeX/명령어가 잘못 쪼개질 가능성 | 중간 | 분리 조건을 `over` 뒤가 숫자인 경우로 제한하고 `\overline` 테스트를 보존한다. |
| `atop` 결합도 같은 결함을 숨기고 있을 가능성 | 낮음~중간 | 동일 패턴 `atop5`를 단위 테스트로 방어한다. |
| 렌더 크기 스케일 변화 | 낮음 | HWP control bbox에 맞춘 scale은 기존 경로를 유지하고 AST만 Fraction으로 바꾼다. |
| picture `current` 크기 적용이 다른 문서 그림을 키울 가능성 | 중간 | `current`가 양수이고 `common`보다 큰 축에만 적용하며, 유틸 단위 테스트와 샘플 SVG로 검증한다. |
