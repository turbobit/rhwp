## 2. 사전 조사 결과

### 2.1 HWPX shape-001.hwpx 내용

두 개의 `<hp:polygon>` (삼각형 4점 path):

| 항목 | 도형 1 (zOrder=0) | 도형 2 (zOrder=1) |
|------|------------------|------------------|
| `<hp:flip horizontal>` | "1" (수평 flip) | "0" |
| `<hp:flip vertical>` | "0" | "0" |
| `<hp:rotationInfo angle>` | 270 | 90 |
| `<hp:rotationInfo centerX/Y>` | 170/425 | 170/425 |
| `<hc:rotMatrix e2>` | 1 | -1 |
| `<hp:treatAsChar>` | 1 (글자처럼) | 1 |

→ 정답지는 두 도형이 거울 대칭 (h-flip + 270도 vs 90도). rhwp 가 flip 또는 회전 처리 누락
시 두 도형이 같은 모양으로 보여 한 쪽이 "180도 잘못" 처럼 보임.

### 2.2 정답지 HWP SHAPE_COMPONENT raw 분석

두 도형 동일 size 40 의 SHAPE_COMPONENT 가짐. 회전 행렬은 TABLE record (size 252) 의 offset 0xA0 에 저장:
- 도형 1: `00 00 F0 3F` = IEEE 754 double **+1.0** (rotMatrix e2)
- 도형 2: `00 00 F0 BF` = double **-1.0** (rotMatrix e2)

HWPX 의 rotMatrix e2 (1 vs -1) 와 정확 정합. 즉:
- HWPX 파서 ✓
- HWPX → HWP 어댑터 ✓
- HWP 파서 (확인 필요)
- **rhwp-studio 렌더링 — transform 미적용 결함 가능**

### 2.3 rhwp 코드 영역 (Explore agent 조사 결과)

| 영역 | 위치 | 상태 |
|------|------|------|
| HWPX polygon 파싱 | `src/parser/hwpx/section.rs:2718` | ✓ 구현 |
| HWPX flip 파싱 | `parse_shape_flip` (line 2059-2080) | ✓ horz_flip/vert_flip 저장 |
| HWPX rotation 파싱 | `parse_shape_rotation_info` (line 2082-2095) | ✓ rotation_angle/center 저장 |

### 2.4 본질 가설

증상 (HWP 도 잘못 + HWPX 도 잘못) 의 공통 영역 = **rhwp-studio 의 polygon 렌더링**.

- `LayerPathOp` 에 transform 필드 없음 → polygon path 의 회전/flip 정보가 렌더링 호출까지 전달 안 됨
- `drawStyledPath()` 가 save/rotate/scale/restore 없음 → 적용해도 path 좌표만 변환되고 회전 무시

비교: `LayerTextRunOp` 는 `transform` 필드 존재 (line 410-413). polygon 도 동일 패턴 필요.
